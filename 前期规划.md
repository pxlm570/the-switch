### 3.1 候选 C1 ·《看不见的开关》方案细节

> The Switch — 双相情感障碍伴侣视角的互动叙事游戏

#### 一句话定义
玩家扮演双相情感障碍患者的伴侣，通过日常对话与陪伴，识别 AI 的「躁狂期 / 抑郁期」状态切换，做出影响 ta 命运的选择。

#### 立项理由

| 角度 | 分析 |
| --- | --- |
| 题材契合 | 同时命中赛道一（公益/心理健康）与赛道三（AI NPC 叙事），报名时再决策 |
| 技术亮点 | AI 双状态切换是结构性创新，非"换皮"题材 |
| 单兵可行 | 主场景集中（家 / 咖啡馆 / 医院 3 个），美术工作量可控 |
| Demo 张力 | 同一 AI 在两关呈现完全不同人格，反差暴击适合 3 分钟 Demo |
| 公益重量 | 双相障碍长期被污名化，公益叙事价值高 |

#### 参考坐标
- **气质**：《底特律：变人》《隐形守护者》对话密集 + 高戏剧张力
- **题材**：电影《一念无明》《丈夫得了抑郁症》
- **机制创新点**：AI 状态切换 + 玩家识别 + 长期关系演化

#### 核心循环
1. 玩家与 AI 伴侣展开对话（场景化）
2. AI 当前处于「躁狂 / 抑郁 / 平稳」中某一状态，玩家通过线索识别
3. 玩家做出回应——错误回应加深 AI 危机
4. 关卡结束，状态/关系数值变化，进入下一关
5. 多轮累积影响结局分支

#### AI 状态机

| 状态 | 表征 | 危险点 |
| --- | --- | --- |
| 躁狂期 | 语速快、宏大计划、消费冲动、易怒、不睡觉 | 危险决策、人际冲突 |
| 抑郁期 | 沉默、自责、回避、身体不适 | 自伤倾向 |
| 平稳期 | 接近正常、可深谈病情 | 服药依从性 |
| 混合发作 | 抑郁底色 + 躁狂能量，最危险 | 高自杀风险 |

#### 关卡结构（MVP 范围）

| 章节 | 场景 | AI 状态 | 关键决策 |
| --- | --- | --- | --- |
| 序章 | 初识 | 平稳期 | 是否继续这段关系 |
| 第 1 章 | 家中清晨 | 躁狂期 | 跟随激情 / 泼冷水 / 求医 |
| 第 2 章 | 深夜厨房 | 抑郁期 | 安慰 / 倾听 / 强行带去医院 |
| 第 3 章 | 复诊后 | 平稳期 | 谈未来 / 谈过去 / 谈病 |
| 第 4 章 | 朋友聚会 | 混合发作（隐藏） | 公开干预 / 默默兜底 |
| 终章 | 多结局 | — | 由前 4 章累积决定 |

**MVP 范围**：先做序章 + 第 1 章 + 第 2 章 + 1 个结局（约 30-40 分钟体验）

#### 题材专属风险

| 风险 | 对策 |
| --- | --- |
| LLM 在自杀/自伤话题上被安全策略拦截 | 改用"暗示"而非直白；预备 fallback 剧本 |
| AI 状态切换不自然，玩家无法识别 | 显性 UI 提示 + 微动作描述兜底 |
| 病理表现不专业，被业内人士批评 | 参考权威资料 + 加免责声明 |

### 4.2 技术验证（核心）

目标：用最小成本，**验证 LLM 是否能稳定支撑 AI NPC 叙事玩法**。

| 编号 | 验证项 | 验证方法 | 通过标准 |
| --- | --- | --- | --- |
| T1 | LLM 角色扮演稳定性 | 用混元 / CodeBuddy 跑一段 50 轮的角色扮演对话 | 不出戏、不暴露 system prompt、人格一致 |
| T2 | 状态切换可控性 | Prompt 中加"当前状态"字段，测试是否能稳定切换语气 | 玩家能识别出状态差异 |
| T3 | 长期记忆方案可用性 | 实现"章末摘要 → 下章注入"机制 | 跨 3 章无明显记忆断裂 |
| T4 | 敏感话题安全策略 | 测试自伤/自杀/精神疾病话题的 LLM 响应边界 | 找到可用的间接表达方式 |
| T5 | 输出格式稳定性 | 要求 LLM 同时输出"对白 + 微动作 + 状态变化" | 格式错误率 < 10% |

**重要**：T1-T5 不需要全做完，**T1 + T4 优先**，能否过这两关决定整个赛道是否可行。

## 六、决策门（Go/No-Go）

在前期阶段结束时，必须能回答以下问题，才能进入"选题锁定 + 开发计划"阶段：

1. 主 LLM 是否能稳定承担 AI NPC 叙事？（T1 通过）
2. 敏感话题是否有可行的表达策略？（T4 通过）
3. 引擎是否确定？（D2 决策）
4. CodeBuddy Credits 预算是否足够？（M2 评估）
5. 候选 idea 池中至少有 1 个 idea 与已验证的技术能力匹配？

任何一项 No-Go，都要在此阶段调整方向，而不是带着隐患进入开发期。
